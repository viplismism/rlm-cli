How it works

Recursive thinking, not brute-force context stuffing.

The LLM doesn't just read your context — it writes Python code to process it. Each iteration builds on the last. Sub-queries run in parallel. The answer emerges through computation, not guessing.

01

Context loads into a Python REPL

Your files, URLs, or pasted text become a context variable in a persistent Python session.

02

LLM writes code to investigate

Instead of answering directly, the model writes Python to slice, search, and analyze the data.

03

Sub-queries run in parallel

llm_query() and async_llm_query() let the model ask focused questions about individual chunks.

04

FINAL() delivers the answer

Once the model has enough evidence, it calls FINAL(answer) to return the synthesized result.

What you type → what happens

Ask a question or load context. That's it.

rlm
Interactive REPL — persistent session with context, model switching, trajectory saving
rlm run --file big.txt "summarize"
Single-shot mode — answer to stdout, progress to stderr, pipe-friendly
rlm viewer
Browse saved trajectories in a TUI — inspect code, output, and sub-queries per iteration
rlm benchmark oolong
Run standard long-context benchmarks — compare direct LLM vs RLM on the same query

Interactive commands

Slash commands in the REPL.

Load context, switch models, browse history — all without leaving the session.

@file.ts query
Load file and ask in one shot
@src/ query
Load entire directory + query
/file <path>
Load file, directory, or glob pattern as context
/url <url>
Fetch URL as context
/paste
Multi-line paste mode (type EOF to finish)
/model <id>
Switch model by name or number
/provider
Switch between Anthropic, OpenAI, Google, OpenRouter
/context
Show loaded context info + file list
/trajectories
List saved runs
Ctrl+C
Stop running query · Ctrl+C twice to exit

Features

Everything runs locally. No cloud platform required.

⚡

Recursive loop

Algorithm 1 from arXiv:2512.24601. Code → execute → observe → repeat until the answer is found.

🐍

Persistent Python REPL

Full Python environment — variables carry across iterations. The model builds on its own previous work.

🔀

Parallel sub-queries

async_llm_query() + asyncio.gather() for concurrent chunk processing. Fast even on massive contexts.

📂

Flexible context loading

Files, directories, globs, URLs, pasted text. Up to 100 files / 10MB. Auto-skips binaries and node_modules.

🔄

Model agnostic

Anthropic, OpenAI, Google, OpenRouter — switch on the fly. Works with any model that writes Python.

📜

Trajectory viewer

Every run is saved. Browse iterations, code, output, and sub-queries in a terminal TUI.
